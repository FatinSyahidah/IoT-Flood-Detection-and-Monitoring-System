# IoT-Flood-Detection-and-Monitoring-System
Early Flood Detection and Monitoring Application based on Internet of Things with Forecasting Model

### **Project Description:**

This project is an IoT-based flood monitoring system that collects real-time environmental data and predicts potential flood conditions. The system integrates sensors, cloud database, web visualization, and a prediction model to provide early flood warnings.

The system monitors temperature, humidity, and water level using IoT sensors connected to a NodeMCU ESP32. Sensor data is transmitted using MQTT protocol and stored in Firebase. A web dashboard built with React visualizes the data in real time. Additionally, a prediction model based on Artificial Neural Network (ANN) is used to forecast water level trends.

### **System Architecture**

## **System Layers**

The system architecture is divided into several layers to separate device communication, data processing, and user interaction.

## **Devices Layer**

The hardware layer consists of sensors connected to a NodeMCU ESP32.

Components used:
1. DHT22 Temperature and Humidity Sensor
2. HC-SR04 Ultrasonic Sensor (water level detection)
3. Buzzer (alert mechanism)

These sensors collect environmental data and send the readings to the NodeMCU microcontroller.

## **Data Transport Layer**

This layer handles communication between the IoT device and the backend services.
1. MQTT Broker manages message transmission.
2. Subscriber receives sensor data from MQTT topics.

The NodeMCU publishes sensor readings through MQTT, and the subscriber processes the incoming data.

## **Database Layer**

Sensor readings are stored in Firebase Realtime Database, which acts as the cloud datastore for the system.

Firebase stores:
1. Temperature readings
2. Humidity readings
3. Water level measurements
4. Device logs
5. Prediction data

## **Analytics Layer**

This layer performs data analysis and prediction.

A Python-based Artificial Neural Network (ANN) model is used to predict future water level conditions based on historical sensor data stored in Firebase.

Prediction inputs include:
1. Temperature
2. Humidity
3. Water level
4. Timestamp

The predicted results are then visualized in the web dashboard.

## **Application Layer**

The application layer provides the user interface and alert system.

**React Dashboard**
1. Displays real-time sensor readings
2. Shows water level graphs
3. Visualizes predicted water level trends

**User Alert System**
1. The system integrates Telegram Bot API to notify users when water levels exceed predefined thresholds.
2. When the subscriber detects dangerous water level conditions, a notification message is automatically sent to users via Telegram.
